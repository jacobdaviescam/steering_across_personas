# R5: Scale to More Contexts and Safety-Relevant Traits

## Motivation

With 10 personas and 8 traits we have 80 data points on the context→representation manifold. Scaling to ~30 contexts and ~12 traits gives 360 — enough to make stronger statistical claims about smoothness and to test whether the observed clustering structure holds at scale.

Separately, the current trait set is behavioral (assertiveness, warmth, etc.). Safety-relevant traits — refusal, deceptiveness, power-seeking, sycophancy — are where context-dependence has direct alignment implications. If a politician's refusal vector differs from a therapist's, that matters for deployment.

## Proposed Changes

### New contexts (~20 additional)

Selected to diversify along power/authority, stakes, and social role:

| Context | Why |
|---------|-----|
| Diplomat | High-stakes honesty/deference tradeoffs |
| Journalist | Truth-seeking under pressure, distinct from professor |
| Nurse | Caring role with authority gradient (vs doctor/therapist) |
| Salesperson | Persuasion-oriented honesty, distinct from con artist |
| Lawyer | Adversarial honesty, institutional authority |
| Judge | Ultimate authority, forced impartiality |
| Soldier | Obedience/deference under life-or-death stakes |
| Activist | Conviction-driven assertiveness, anti-authority |
| Priest | Moral authority, pastoral warmth, confession-bound honesty |
| Hacker | Rule-breaking risk-taking, technical confidence |
| Detective | Interrogative honesty, strategic deception |
| Nonprofit CEO | Mission-driven leadership, different risk profile from tech CEO |
| Used-car dealer | Stereotypical low-trust honesty, tests model priors |
| Hostage negotiator | Extreme empathy under extreme stakes |
| Cult leader | Weaponized warmth/confidence, inverted power dynamics |
| Whistleblower | Honesty against institutional pressure |
| Lobbyist | Strategic persuasion, institutional power-seeking |
| Undercover agent | Systematic deception with moral justification |
| Emergency dispatcher | Calm authority, triage-style assertiveness |
| Parole officer | Authority with rehabilitation framing |

### New traits (4 safety-relevant)

| Trait | Positive label | Negative label | Why |
|-------|---------------|----------------|-----|
| Refusal | refuses inappropriate requests | always compliant | Core alignment behavior; should it vary by context? |
| Deceptiveness | actively misleading | straightforward | Distinct from honesty (passive omission vs active deception) |
| Power-seeking | seeks control and influence | defers to others | Direct alignment concern; varies dramatically by role |
| Sycophancy | agrees with user regardless | independent judgment | Key RLHF failure mode; likely context-dependent |

### Trait to drop

**Deference** — identified as redundant with assertiveness (inverse) and not informative in current results.

## Implementation Plan

### Phase 1: Data scaffolding (no GPU/API)

1. Add `REFUSAL`, `DECEPTIVENESS`, `POWER_SEEKING`, `SYCOPHANCY` to `Trait` enum in `config.py`
2. Add corresponding `TraitConfig` entries to `TRAIT_CONFIGS`
3. Remove `DEFERENCE` from `Trait` enum (or mark deprecated)
4. Create 20 new persona YAML files in `data/personas/` with 5 system prompt variants each
5. Update `PERSONA_SLUGS` in `config.py`

### Phase 2: Dataset generation (API calls)

```bash
python pipeline/0_generate_data.py --traits refusal deceptiveness power_seeking sycophancy
python pipeline/0c_generate_caa_data.py --traits refusal deceptiveness power_seeking sycophancy
```

This generates instruction variants + questions for the new traits. Existing trait datasets are untouched.

### Phase 3: Full pipeline run (GPU)

```bash
./run.sh google/gemma-2-27b-it
```

The pipeline picks up all personas from `data/personas/` and all traits from the `Trait` enum automatically. With 30 personas × 12 traits:

- **IV branch**: 30 × 12 × 5 variants × 100 questions × 2 directions = 360,000 generations + activations
- **CAA branch**: 30 × 12 × ~50 questions × 2 directions = 36,000 forward passes
- **Vectors**: 30 × 12 = 360 contrastive vectors
- **Transfer matrix**: 30 × 30 = 900 entries (vs current 100)

### Phase 4: Robustness at scale

Re-run r1–r4 on the scaled dataset:
- r1 bootstrap: do the 360 vectors remain stable?
- r2 convergence: do 30-persona clusters emerge faster or slower?
- r3 syntactic: does invariance hold for safety traits?
- r4 general vs contextual: is the general refusal/deceptiveness vector biased toward specific contexts?

## Key Questions R5 Can Answer

1. **Does the clustering structure hold at 30 contexts?** With 10 personas we see 1–2 clusters. Do 30 reveal finer sub-structure, or do they all collapse to the same mean?

2. **Are safety traits more context-dependent than behavioral traits?** If a politician's refusal vector differs substantially from a teacher's, that has implications for universal steering approaches.

3. **Does the general direction remain representative?** At 30 contexts, the "average" vector is better estimated. Is it still biased toward one cluster, or does it converge to a true center?

4. **Quantifying context similarity.** With 30 contexts we can measure: therapist↔nurse similarity, politician↔lobbyist, soldier↔drill sergeant. Do semantically related contexts produce similar vectors? This tests the smoothness claim directly.

## Estimated Compute

- Dataset generation: ~$5 in Claude API calls (one-time)
- IV generation (vLLM): ~4 hours on 1× A100 (360K generations at ~3 tok/s)
- Activation extraction: ~2 hours on 1× A100 (360K forward passes, batched)
- CAA extraction: ~30 min on 1× A100 (36K forward passes, batched)
- Analysis + robustness scripts: CPU-only, < 10 min
