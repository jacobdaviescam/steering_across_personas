# R7: Variant Convergence (Instruction-Variant Axis)

## Motivation

R2 measured convergence along the **question axis**: how many question samples are needed before the steering vector stabilizes? But there's a second axis — **instruction variants**. We use 5 semantically-equivalent instruction pairs per trait (e.g., "be assertive" vs "be very assertive and direct"). R3 showed that within-persona, across-variant similarity is only 0.659 — meaning the 5 variants produce meaningfully different per-variant vectors.

Jacob and Nathaniel both asked: if you build a vector from just 1 variant, vs 2, vs all 5, how does it converge? This isolates **syntactic robustness** from **sample-size robustness** (which r2 already covers).

This matters for practitioners: do you need 5 carefully worded instruction pairs, or does 1 suffice?

## What It Measures

For each persona × trait:
1. Compute per-variant vectors: v_1, v_2, ..., v_5 (each using all ~100 questions but only 1 instruction variant)
2. Compute cumulative vectors: V_1 = v_1, V_{1:2} = mean(v_1, v_2), ..., V_{1:5} = mean(v_1, ..., v_5) = full vector
3. Measure cosine(V_{1:k}, V_{1:5}) for k = 1, 2, 3, 4
4. Also measure stability: for each k, sample k variants randomly (C(5,k) combos) and compute spread

## Implementation

### Inputs
- `outputs/{model}/activations/{persona}_{trait}_{pos|neg}.pt` — existing activation files
- Need to know which activations correspond to which variant. Check whether variant index is stored in the activation metadata or can be reconstructed from the response JSONL files.

### Script: `pipeline/r7_variant_convergence.py`

```python
# Pseudocode
for persona, trait in all_pairs:
    pos_acts = load activations, group by variant index
    neg_acts = load activations, group by variant index
    
    for k in [1, 2, 3, 4, 5]:
        for combo in combinations(range(5), k):
            v_k = mean(pos_acts[combo]) - mean(neg_acts[combo])
            cosine_to_full = cosine(v_k, v_full)
        
        results[persona][trait][k] = {
            "mean_cosine": mean across combos,
            "std_cosine": std across combos,
            "n_combos": C(5,k)
        }
```

## Key Questions

1. How many instruction variants are needed for a stable vector? If 1–2 suffice, the extraction process can be 3–5× cheaper.
2. Is the variant axis noisier or smoother than the question axis? (Compare convergence rate to r2.)
3. Are some variants systematically better than others, or is it random noise?
