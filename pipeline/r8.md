# R8: General Direction Cluster Bias

## Motivation

Jacob raised a specific question: when we compute the "general" steering vector for a trait (mean across all personas), is it equidistant from all persona vectors, or is it biased toward one cluster?

With 10 personas, R4 found only 1 cluster per trait, so the general direction trivially equals the cluster centroid (cosine ≈ 1.0). This experiment can't produce interesting results until we scale to 30+ contexts (R5). But the analysis framework should be ready.

The alignment implication: if the general honesty direction is closer to "farmer honesty" than "politician honesty," then applying the general vector preferentially steers in one context's direction. A truly universal vector should be equidistant from all contexts.

## What It Measures

1. **Distance asymmetry**: For each trait, compute cosine(general, persona_i) for all personas. Test whether these distances are uniform (null hypothesis) vs. clustered (alternative). Use Levene's test or coefficient of variation.

2. **Cluster-centroid bias** (requires R5 scale): If personas cluster into k>1 groups, compute cosine between the general vector and each cluster centroid. Is the general vector equidistant from all clusters, or pulled toward the largest one?

3. **Leave-one-out sensitivity**: For each persona, compute the general vector excluding that persona. How much does removing one context shift the general direction? Identifies disproportionately influential contexts.

4. **Weighted vs. unweighted general**: Compare the simple mean (equal weight per persona) to a reweighted mean (e.g., inverse-variance weighting, or downweighting outliers). Does the weighting scheme matter?

## Implementation

### Script: `pipeline/r8_cluster_bias.py`

```python
# Pseudocode
for trait in traits:
    vectors = {p: load_vector(p, trait) for p in personas}
    general = mean(vectors.values())
    
    # Distance asymmetry
    cosines = {p: cosine(general, v) for p, v in vectors.items()}
    cv = std(cosines) / mean(cosines)  # coefficient of variation
    
    # Leave-one-out
    for p in personas:
        others = [v for p2, v in vectors.items() if p2 != p]
        general_loo = mean(others)
        loo_shift[p] = cosine(general, general_loo)
    
    # Cluster bias (meaningful at N >= 20)
    if n_personas >= 20:
        labels = cluster(vectors, method="agglomerative", threshold=t)
        for cluster_id in unique(labels):
            centroid = mean(vectors in cluster)
            bias[cluster_id] = cosine(general, centroid)
```

## Can Run Now?

**Partially.** The leave-one-out and distance asymmetry analyses work with 10 personas. Cluster bias analysis is only meaningful at 30+ personas (post-R5). Implement both paths with a conditional on N.

## Relationship to R4

R4 already computes `general/mean_cosine` per trait and identifies `most_different` persona. R8 goes deeper: instead of just reporting the max outlier, it tests whether the distribution of distances is uniform, measures influence of individual contexts, and (at scale) tests cluster-level bias.

## Key Questions

1. Is any single persona disproportionately influential on the general direction?
2. At 30 contexts, does the general vector sit equidistant from all clusters, or is it pulled toward a "default" context?
3. Does reweighting (e.g., downweighting outlier personas) produce a more robust general direction?
